Hotel Booking Case Study

The goal of this project was to create a mobile booking app for a new hotel company. They were hoping for a simple and accessible process that would allow users to quickly find and reserve hotel rooms.

Intro

Research

Competitive Benchmark

The competitive benchmarking process was helpful in getting to know the current hotel booking landscape. It yielded a few key findings:

  • Uncluttered interfaces were helpful
  • Minimizing form length was a positive
  • Encountering frequent sign-up calls to action was frustrating
  • Well-placed trip summaries were helpful
  • Date selection was often a point of confusion

Usability Testing

Insights from three different usability tests were collected. Two of the tests were part of the course materials, while the third was conducted as a course project. The insights collected included:

  • Users love a good deal
  • Brand-name recognition is helpful
  • Videos/imagery help people get excited
  • Calendar selection could be tricky (slow load, etc)
  • Photos and copy important

Analysis

Affinity Diagram

The affinity diagram stage synthesized the findings from the research stage, including:

  • Reasons for choosing a brand
  • UI elements (the date selector, iconography, etc)
  • Filtering

Customer Journey Map

A customer journey map helped to understand where the product fits within the larger decision process as well as to identify various emotions through the hotel booking journey:

  • Customers often think and research about the trip before committing to a particular hotel
  • The beginning and end of the process are typical the most enjoyable for users
  • The location search can be frustrating if poorly implemented

Flow Diagram

A flow diagram was created to determine the primary user flow through the app. This was kept similar to competitors studied in order to follow established conventions.

Design

Sketches

Sketches each of the screens in the app as a rough draft for the final product with special attention to issues that were noted previously such as:

  • Added a date selector with a vertical scroll to the next month to keep selection as intuitive as possible
  • Created two large tabs to allow for map and filter views on the results page
  • Kept the checkout form as short as possible
  • Included TripAdvisor reviews widget to add trust as this is a new hotel brand

Prototyping

The prototyping stage included a medium-fidelity prototype as well as developer handoff materials. The medium-fidelity prototype was chosen because it would provide a close representation of the final product for testing while still having a fairly quick completion time.

Prototype

The prototype was set up to test the following scenario:

  1. Select Baltimore as destination
  2. Set dates to be June 10 - 16
  3. Edit the reservation to be for 2 adults
  4. Add a AAA discount
  5. Set distance to be up to 5 miles
  6. Set price to be $75 - $125
  7. Make sure the breakfast is complimentary
  8. Select a queen room at Hotel on the Inner Harbor
  9. Check out as member
